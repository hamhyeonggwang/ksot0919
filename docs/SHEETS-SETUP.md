# Google 스프레드시트 연동 설정 가이드

신청 폼(보수교육·대학원생 워크숍 등) 제출 내용을 구글 스프레드시트에 자동 기록하는 설정 절차.
**한 번만 하면 되는 작업이며, 약 5분 소요됩니다.**

## 연동 구조

```
신청 폼 제출 → Google Apps Script(웹 앱) → 스프레드시트에 행 추가
```

- 접수 스프레드시트: https://docs.google.com/spreadsheets/d/1ULBG3ILsMKq5zzOqZyADBv_26vhjBm0ta-7Hw0VdDsc/edit
- 시트는 자동 생성됨: `보수교육_임상가` / `보수교육_학생` / `대학원생_워크숍` /
  `포스터_발표` / `구두논문_발표` / `캡스톤_디자인` / `기타_접수`
- 헤더(1행)도 자동으로 채워지므로 시트를 미리 만들 필요 없음

## 설정 절차

### 1. Apps Script 열기

1. 위 스프레드시트를 연다 (h2g0614@gmail.com 등 **운영 계정**으로 로그인)
2. 메뉴 **확장 프로그램 > Apps Script** 클릭
3. 편집기에 기본으로 있는 `function myFunction() {...}` 코드를 모두 지운다

### 2. 코드 붙여넣기

1. 이 저장소의 **`register/apps-script.gs`** 파일 내용 전체를 복사해 붙여넣는다
2. 저장 (💾 아이콘 또는 Cmd+S)

### 3. 웹 앱으로 배포

1. 우측 상단 **배포 > 새 배포** 클릭
2. ⚙️(유형 선택) > **웹 앱** 선택
3. 설정:
   - 설명: `학술대회 접수` (자유)
   - 다음 사용자 인증 정보로 실행: **나**
   - 액세스 권한이 있는 사용자: **모든 사용자** ← 중요!
4. **배포** 클릭
5. "액세스 승인" 창이 뜨면 → 계정 선택 → "고급" → "(프로젝트명)(으)로 이동" → 허용
   (직접 만든 스크립트이므로 안전함)
6. 발급된 **웹 앱 URL**(`https://script.google.com/macros/s/.../exec`)을 복사

### 4. 웹페이지에 URL 연결

`register/register.js` 6번째 줄의 `SCRIPT_URL`에 복사한 URL을 붙여넣는다:

```js
const SCRIPT_URL = 'https://script.google.com/macros/s/XXXXXXXX/exec';
```

저장 후 배포(GitHub Pages/Vercel)하면 끝.

### 5. 동작 확인

1. 브라우저에서 웹 앱 URL을 직접 열었을 때 "정상 동작 중입니다" 문구가 보이면 배포 성공
2. 신청 페이지에서 테스트 제출 → 스프레드시트에 행이 추가되는지 확인
3. 테스트 행은 삭제

## 코드를 수정했을 때 (재배포)

Apps Script 코드를 수정한 경우 저장만으로는 반영되지 않음:

1. **배포 > 배포 관리** 클릭
2. ✏️(수정) > 버전: **새 버전** 선택 > **배포**
3. URL은 그대로 유지되므로 register.js는 수정 불필요

## 자주 묻는 문제

| 증상 | 원인 / 해결 |
|------|-------------|
| 제출해도 시트에 안 들어옴 | 배포 시 액세스 권한이 "모든 사용자"인지 확인 |
| 코드 수정이 반영 안 됨 | "배포 관리 > 새 버전"으로 재배포해야 함 |
| 권한 오류 | 스프레드시트 소유 계정과 Apps Script 실행 계정이 같은지 확인 |
| 다른 시트에 기록하고 싶음 | `apps-script.gs` 상단 `SPREADSHEET_ID` 교체 |
